## 16. Build order

A vertical slice before breadth, so the contract is proven end-to-end early. Cores land **CLI-first** — a bare short prints `help` until its TUI exists (§7.3) — so the chrome arriving mid-list is what the grammar already allows, not a debt it runs up. The repo is public and CI green from the first commit: licences, hygiene, and the per-PR checks (§15.1–§15.2) precede this list rather than closing it, `dist plan` alone excepted — dist ignores a package that defines no binary, so a workspace of libraries gives it nothing to plan and that check joins at step 1 (§15.2).

1. **Pantheon + `pan`** — the spine (addressing, resolution, the record envelope, the `Core` substrate, write-time validation — §5) and the thin bin over it (§5.5, §14). The spine is a library, so nothing exercises it until its bin exists: `pan new` mints the tree a core will later write into (§7.3), and `pan tree` / `resolve` / `validate` are how it is read back before any core can. First binary in the workspace, so `dist-workspace.toml` and CI's `dist plan` land here (§15.2–§15.3).
2. **Annales** — the simplest core, one hand-named `log` series (§8.6); proves I1, the Series shape, and the contract.
3. **Album** — the first partitioned register (§8.1); proves the second shape, `core:slug` refs and the resolver's filename path (§5.0), and the rename cascade (§5.4).
4. **Pensum** — the nameless `task` series (§8.5), minted by a node's first task rather than by `-c`: the determined-name path Annales' hand-named log never reaches (§7.3). It is also the file a detached hook and a hand contend for, which is what the record lock exists for (§6.4), and the tasks Atrium's day is made of (§12).
5. **Tabella** — the third shape (§6.1, §8.7). A Document core declares no tokens at all (§7.1), so it is the one file→core mapping resting on extension alone, and the one that makes `pan doctor`'s totality claim demonstrable rather than merely stated (§5.5). Its `edit` is the editor form's home ground (§7.3).
6. **Porticus + Tessera + Atrium** — the first screen; proves derived-out rendering (I1) and the relayed human write (I2, §12), and gives the cores and `pan` their TUIs at once. The chrome's design is `PORTICUS-SPEC.md`'s (§11.1). The slice closes here: three shapes below, one contract across them, one screen above.
7. **Mappa, Rationes, Fasti** — the cores the slice did not need, against the proven contract. Fasti brings the two-token core (a partitioned `span` beside an `event` series, §8.4), Rationes the determined-name series (§8.3).
8. **Auspex** — the first rule, `plan` before `run`; proves the propose protocol, the capability grant, and the one reactive writer (I2). It lands last of the writers because a grant's vocabulary is `core@home` (§9.2): with every core built, the capability surface is written once against the whole of it rather than revisited as each target appears. Steps 1–7 are themselves the `aus`-not-on-`PATH` case (§9.4), so the hook spawn every core carries is exercised as a no-op long before there is anything to wake.
9. **Speculum, Studium** — the lenses whose reach is the point (§12): both compose across cores, and Studium's GPA folds Fasti spans (§8.4), so neither is buildable before step 7.
10. **Releases** — `release-plz`, the per-crate tags, and the first tagged `dist` run (§15.3–§15.4), once an app is worth installing.
