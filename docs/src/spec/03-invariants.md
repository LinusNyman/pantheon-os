## 3. Design invariants

- **I1 — Samples in, present derived out.** Anything measured or logged is a reading keyed by date, accumulated into a series; every present value is *derived* — the reading at the latest key, or a Tessera fold — never stored as a `current_*` field. Readings are correctable in place: fixing a wrong value rewrites that keyed line, it does not append a second. Entity registers (people, places, holdings) follow the same rule, keyed by slug — one mutable record per entity, corrected in place. The stored record is the only truth; the present is always read from it.
- **I2 — One reactive writer.** No component may *originate* a write on its own; only **Auspex** turns a read into an automatic write. A write must be authored by a hand (human or LLM) or by Auspex. This scopes to *authorship*, not to which binary the keystroke passed through: a lens relaying a human's command to a core's write verb is the human's hand acting (direct ingress, §7.2), and is allowed; a lens writing because a reading changed is not.
- **I3 — The path is the home.** Each record has exactly one home, and that home *is* its file's location; no `home` field is stored. Aboutness is location: a record homes at what it is *about* — not at the activity or context that produced it, so **provenance is an edge, never a parent** — and lives *in* that node rather than pointing at it. The only reference to another record is by `core:slug` — an entity, a document, a task, or a named series (location-independent, survives a re-home).
- **I4 — One contract.** Every core exposes a CLI that emits JSON. That single surface serves humans, lenses, Auspex, and LLMs. The human tool and the machine interface are the same thing.
- **I5 — References, not dependencies.** A core stores another core's entity as a bare ref; it neither imports nor reads the other core. Pantheon resolves at read time. Hub-and-spoke, never a mesh: **no component points sideways** — no core depends on another core, and no package on its peer. Reach *across* cores is a lens's alone (§12): a core reading a peer's JSON off `PATH` links nothing and still points sideways, which is why the prohibition is on the reading and not merely on the link.
- **I6 — Legible storage.** No central store, no opaque IDs. Every record is a plain file under its home node — entities as one `.json` each and series as `.jsonl`, both in the node's meta dir; documents as loose Markdown in the node dir — keyed by a human token (slug or `YYMMDD`), hand-editable at the source. Structure is enforced on write and re-checked on next read or on demand, never by something watching (§18), so direct hand or LLM edits are a first-class path.
- **I7 — The reality test governs depth.** A node is real only if distinct things are filed there and it is reviewed apart. A blank sub-level is a finished answer, not a gap.
- **I8 — Three hands: human, LLM, code.** Every act on the data — read, add, edit, move, decide — must be doable directly by a human, by an LLM on their behalf, or by deterministic code, with all three equally fluent and free to hand off mid-task: same files, same addresses, same JSON contract, same validation. I8 is the tie-breaker when other choices are balanced.
- **I9 — Relationships are edges, not nesting.** An entity is filed once, by what it *is*; every relationship it bears — membership, association, provenance, "relates to" — is a reference (`core:slug`), never a tree nesting. Nest a node beneath another only when it is *part of the substance of* it; reference when it merely *belongs to*, *relates to*, or *came from* it. Corollary: **never reproduce one branch's structure inside another** — no people-tree inside a project, no project-tree inside people; the cross-cut is a query over edges, not a duplicated subtree. This is why `core:slug` is the only reference form a *record* has, and why no record points at a node (§5.4, §18): membership in a family, club, or org points at the group *entity*, not at a directory. (One code-valued token on disk does name a node — a rule's `writes=` capability, §9.2 — but a grant of authority is not a record's reference, and it is cascaded on a re-home like the code it is, §10.1.)
